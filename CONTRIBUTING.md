# 기여 가이드

student-kit에 기여해주셔서 감사합니다! 🎉

---

## 어떻게 기여할 수 있나요?

### 1. 새 슬래시 커맨드 제안
학생 작업 중 자동화되면 좋을 것이 있다면:
- [Issues](https://github.com/gocks77777/student-kit/issues) 에 아이디어 제안
- 또는 직접 PR 보내기

### 2. 기존 커맨드 개선
- CEO 모드 질문이 부족하다면 추가
- 워크플로우를 더 효율적으로 개선
- 한글 별칭 추가/개선

### 3. 버그 리포트
- 동작하지 않는 부분 발견 시 Issue 생성
- 환경 정보 (OS, Claude Code 버전) 포함

### 4. 문서 개선
- README, INSTALL 등 문서가 헷갈리면 개선
- 사용 예시 추가
- 한글 번역 / 영문 번역

---

## 슬래시 커맨드 작성 가이드

새 슬래시 커맨드를 만들 때 다음 원칙을 지켜주세요:

### 1. CEO 모드 필수
모든 슬래시 커맨드는 `Phase 0: CEO Challenge` 를 가져야 합니다.
`docs/ceo_principles.md` 를 참고하세요.

### 2. 파일 구조
```markdown
---
description: 한 줄 설명
argument-hint: <인자 형식> (있는 경우)
---

# /command-name - 짧은 설명

> CEO 모드 적용 — `docs/ceo_principles.md` 참고

## Phase 0: CEO Challenge (질문 N개)
[2~3개 날카로운 질문]

## 응답 처리
[Case A/B/C 처리]

## Phase 1~N: 본 작업
[워크플로우]

## 출력 형식
[표/마크다운 예시]
```

### 3. 한글 별칭 추가
새 영문 커맨드를 추가했다면, 한글 별칭도 함께 추가:

```markdown
---
description: 한 줄 설명 (한글 별칭)
---

`commands/{영문커맨드}.md` 와 동일하게 동작해.

핵심 질문 (Phase 0):
1. ...
2. ...
```

### 4. 톤 일관성
- 한국어 반말
- 짧고 직설적
- "당신" → "너"
- "~합니다" → "~해"
- 친구 같은 말투, 봐주지 않음

### 5. 일반화
- 특정 사용자 정보를 하드코딩하지 말 것
- "사용자의 CLAUDE.md 에서 ~ 추출" 같은 형식
- 누가 써도 동작해야 함

## PR 절차

1. Fork
2. 새 브랜치 생성: `git checkout -b feat/new-command`
3. 변경사항 commit
4. Push: `git push origin feat/new-command`
5. PR 생성, 다음 정보 포함:
   - 어떤 학생 작업을 자동화하는지
   - CEO 모드 질문 예시
   - 사용 예시

---

## 코드 리뷰 기준

- ✅ CEO 모드 적용됐는지
- ✅ 한글 별칭 있는지
- ✅ 일반화돼 있는지 (개인 정보 X)
- ✅ 출력 형식 명확한지
- ✅ 한국어 톤 일관성
