# /jobs - 잡 헌팅 통합 (CEO 모드)

> CEO 모드 적용 — `commands/_ceo_principles.md` 참고

---

## Phase 0: CEO Challenge (질문 3개)

잡 검색 돌리기 전에 3가지 답해줘.

> 💡 빠른 모드: 사용자가 "빠르게" / "fast" 입력 시 → Q2만 (1순위 강제 선택)

### 질문 1: WHY
> **"너 진짜 취업하고 싶어?"**
>
> 부모님 때문 아니야? 동기들 때문 아니야?
> 너 자신한테 솔직하게.

### 질문 2: 강제 1순위 선택
> **"1순위 회사 유형 1개만 골라."**
>
> 자유롭게 답해 — 카테고리 정해진 거 없음.
> 다 좋아하는 건 아무것도 진심이 아니라는 뜻이야.
>
> (참고용 예시: 리모트 / 외국계 / 한국 대기업 / 한국 스타트업 / 프리랜서 /
> 정부기관 / 연구소 / 비영리 / 본인만의 카테고리 — 위에 없어도 OK)

### 질문 3: 인버전
> **"합격해서 1년 안에 그만둘 회사야?"**
>
> 그럼 왜 시작해? 시간 낭비 아니야?

## 응답 처리 (의미 기반)

### Category A: 3개 다 명확 + 1순위 단호
**의미 패턴**: 본인만의 답, 단호함, Q1과 Q2 일관

→ ✓ Phase 1로

### Category B: 우선순위 없음 / 모호함
**의미 패턴**:
- 여러 카테고리 다 좋다는 답변
- "딱히 상관없어" 류
- 회피성 답변

(예시 — 참고용: "다 괜찮아", "어디든 좋아", "별로 가리지 않아")

→ "그게 문제야. 우선순위 없으면 모든 회사에 70점 자소서 내고 다 떨어져.
   진짜 1개만 골라. 못 고르면 잠깐 멈춰."
→ 최대 1회 재질문, 그래도 못 정하면 `/고민 진로 우선순위` 추천

### Category C: 회피성 답변
**의미 패턴**:
- "그냥 알려줘" 류
- 짜증/귀찮음

→ "ㅇㅋ 근데 우선순위 없는 잡 검색은 시간 낭비. 그래도 진행할까?"

## Phase 1: 사용자 프로필 로드 (캐시 우선)
- `CLAUDE.md` 가 컨텍스트에 있으면 재읽기 X (토큰 절약)
- 없으면 Read (전공, 학년, 진로, 기술 스택)
- `plan.md` 는 Grep으로 "회사", "직무", "목표" 부분만 추출
- 깃허브 프로필 있으면 최근 프로젝트 참고
- **Phase 0의 1순위** 를 검색 우선순위로 설정

## Phase 2: Job Finder 실행 (있는 경우)
- 사용자의 Job Finder 스크립트가 CLAUDE.md에 정의돼 있는지 확인
  (예: `python ~/jobfinder/find_jobs.py`)
- 있으면 실행, 없으면 Phase 3로

## Phase 3: 새 공고 수집 (병렬 검색)
WebSearch 동시 실행 (sequential X):
- "{Phase 0의 1순위 카테고리} {사용자 직무} 채용 2026"
- 한국 사이트: 사람인, 잡코리아, 원티드, 점핏
- 해외 사이트: LinkedIn, We Work Remotely (리모트면)

> 💰 토큰 절약: 1세션에 같은 검색 이미 했으면 캐시 사용

## Phase 4: 매칭 점수 계산
각 공고에 대해 1~10점:
- 기술 스택 매칭 (4점)
- 경력 요구사항 (3점)
- 지역/리모트 가능성 (2점)
- 회사 인지도/안정성 (1점)
- **Phase 0의 1순위와 일치하는지 가중치 +2점**

## Phase 5: Top 5 출력
점수 순 정렬, 표 형태:
```
| 순위 | 회사 | 직무 | 점수 | 마감 | Phase 0 매칭 | 한 줄 평가 |
```

## Phase 6: 액션 추천
Top 1~2 회사에 대해:
- "지원할만 함" / "검토 필요" / "패스 추천" 판단
- 지원하기로 한 회사는 `/coverletter [회사명]` 추천

## 주의사항
- 사용자 정보가 부족하면 먼저 CLAUDE.md 업데이트 제안
- 한국어로 답변
- Phase 0의 답변과 일치하지 않는 추천은 표시하지 말 것
