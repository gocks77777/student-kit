## Phase 0: CEO Challenge (질문 2개)

논문 요약 전에 2가지만.

> 💡 빠른 모드: "빠르게" / "fast" 입력 시 → Q1만, 빠른 모드 자동 선택

### 질문 1: 동기 명확화
> **"시험 때문이야 아니면 진짜 알고 싶어?"**
>
> 자유롭게 답해. 다음 모드 중 하나로 처리됨:
> - **시험 모드** (15분, 핵심만)
> - **깊은 모드** (1시간, 코드까지)
> - **본인 모드** (예: "리서치용 30분", "강의 발표용 5분")
>
> 어느 거야? 솔직하게.

### 질문 2: SO WHAT
> **"이 분야에서 너의 5년 후 모습은?"**
>
> "그냥 한 번 읽어봐" 면 시간 낭비야.
> 너의 진로/연구랑 어떻게 연결되는지 말해.
> 연결 안 되면 → 시험 모드로 갈게.

## 응답 처리 (의미 기반)

### Category A: 명확한 답변
**의미 패턴**: 모드 명시 + 진로 연결 1줄

→ 해당 모드로 Phase 1 진행

### Category B: 모호한 답
**의미 패턴**: "그냥", "한번", 목적 불명

→ 빠른 모드 기본값으로 진행 (시간 낭비 방지)

### Category C: 진짜 관심이라 주장하지만 연결 없음
**의미 패턴**:
- "관심 있어" 답변
- 진로/연구와 연결 못 만듦

→ "연결 못 만들면 시험 모드 추천. 깊은 모드는 시간 낭비될 수 있어."

## Phase 1: 논문 접근 (효율적)
- $ARGUMENTS가 URL이면 WebFetch (1번 호출)
- 논문명이면 Google Scholar 검색 → PDF 링크 1개만 → WebFetch
- arXiv, OpenReview, CVPR/ICLR/NeurIPS 우선

> 💰 토큰 절약: 논문 전체 fetch 후 필요한 섹션만 추출 (Read X, 메모리에 유지)

## Phase 2: 핵심 정보 추출
- 제목, 저자, 학회, 출판년도
- Abstract 핵심 한 줄
- 인용 수 (가능하면)
- 주요 contribution 3~5가지

## Phase 3: 학생 친화 요약 작성

파일명: `paper_summary_{논문약칭}.md`

### 구조 (Phase 0 모드별로 다름)

#### 빠른 모드 (시험 대비, 15분)
```markdown
# {논문 제목} 시험용 요약

## 한 줄 정리
{초등학생도 이해할 한 줄}

## 시험에 나올 핵심 5가지 ★
1. ...
2. ...
3. ...
4. ...
5. ...

## 자주 나오는 질문 + 답변
Q1. ...
A1. ...

## 핵심 그림/수식
- Figure X: ...
- Equation Y: ...
```

#### 깊은 모드 (진로 연결, 1시간)
```markdown
# {논문 제목} 학생용 요약 (4페이지)

## 한 줄 정리
{초등학생도 이해할 한 줄}

## 1페이지: 배경 + 문제
- 이전에 뭐가 문제였나?
- 왜 이 논문이 필요했나?
- **비유/일상 예시** 로 설명

## 2페이지: 핵심 아이디어
- 무엇을 새로 제안했나?
- 어떻게 동작하나?
- 다이어그램/수식이 있다면 평어로 풀어서 설명
- **일상 비유** 활용

## 3페이지: 실험 결과
- 무엇을 측정했나?
- 결과 표/그래프 의미
- 이전 방법 대비 얼마나 좋아졌나?

## 4페이지: 영향 + 너의 진로 연결
- 이 논문 이후 어떻게 발전했나?
- **너의 5년 후 목표와 어떻게 연결되는지** (Phase 0 답변 기반)
- 후속 논문 추천
- 코드 구현체 GitHub

## References
- {scholar.google.co.kr 형식}
```

## Phase 4: 시험 대비 강조 (★ 표시)
- 핵심 수식/공식
- 그림 번호 (Figure X 참고)
- 자주 헷갈리는 개념
- 비교 표 (이전 방법 vs 새 방법)

## 주의사항
- 어려운 영어 용어는 한국어 + 원어 병기
- 비전공자도 이해할 수준 (단, 컴퓨터공학 기초는 가정)
- 한국어, 반말 OK
