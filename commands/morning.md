# /morning - 데일리 브리핑

> CEO 모드 적용 — `commands/_ceo_principles.md` 참고

## Phase 0: CEO Challenge (질문 1개)

오늘 작업 정리하기 전에 하나만 물어볼게.

> **"오늘 만약 1가지만 한다면 뭐를 해야 가장 의미 있어?"**
>
> 나머지는 다 부수적이야.
> 이거 못 정하면 오늘도 그냥 흘러갈 거야.

답해. 그 1가지 중심으로 오늘 정리해줄게.

### 응답 처리
- **명확한 1가지 답**: ✓ Phase 1로
- **여러 개 답**: "다 중요해 보여? 정확히 하나만 골라. 못 고르면 오늘 우선순위 안 잡혀."
- **모르겠다**: "그게 진짜 문제야. 일단 plan.md 같이 보면서 찾자."

---

## Phase 1: 사용자 컨텍스트 로드
- 현재 디렉토리의 `CLAUDE.md` 읽기
- `plan.md`, `todo.md`, `deadlines.md` 등이 있으면 읽기
- 위 파일이 모두 없으면 사용자에게 어떤 컨텍스트 파일을 쓰는지 묻기

## Phase 2: 마감 임박 체크
- 컨텍스트 파일에서 날짜/마감 정보 추출
- **오늘 기준 7일 이내** 마감 모두 표시
- D-day 형식으로 (D-3, D-7 등)
- 마감 7일 이내가 없으면 "이번 주 임박 마감 없음" 표시

## Phase 3: 어제 작업 요약
- `git log --since="yesterday" --oneline` 실행
- 주요 변경사항 1~3줄 요약
- 커밋이 없으면 "어제 커밋 없음"

## Phase 4: 오늘 우선순위 Top 3
- **Phase 0에서 답한 "1가지" 를 1순위로**
- 마감 임박도 + 사용자 목표 기반으로 2~3순위
- 각 작업별 예상 소요 시간 표시

## Phase 5: 첫 작업 1개 추천
- 30분~1시간 안에 끝낼 수 있는 것
- 가장 ROI 높은 것
- "지금 바로 시작하면 좋을 작업" 으로 표현

- ...
