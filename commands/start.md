# /start - student-kit 첫 셋업 (CEO 모드)

> CEO 모드 적용 — `commands/_ceo_principles.md` 참고

처음 student-kit 사용하는 사람을 위한 대화형 셋업.
사용자에게 한 번에 한 질문씩 묻고, 답변 받아서 CLAUDE.md 파일 자동 생성.

---

## Phase 0: 환영 메시지

```
👋 환영해. student-kit 처음이지?

이건 일반 도구랑 달라.
"빠르게 답 주는" 도구가 아니라
"네 답변을 거부하고 진짜 답을 끌어내는" 도구야.

처음 5분만 투자해. 그 후 모든 슬래시 커맨드가
너 전용으로 동작해.

준비됐어? 아니면 나중에 할래?
```

→ "준비됐어" / "ㅇㅇ" / "ㄱㄱ" 류 답변이면 Phase 1
→ "나중에" 면 종료

---

## Phase 1: 기존 파일 체크

현재 디렉토리에 `CLAUDE.md` 가 있는지 확인.

### Case A: 없음
→ "ㅇㅋ. 새로 만들자." → Phase 2

### Case B: 이미 있음
```
⚠️ CLAUDE.md 이미 있어.

선택해:
1. 덮어쓰기 (기존 내용 사라짐)
2. CLAUDE.md.new 로 저장 (안전)
3. 취소
```

---

## Phase 2: 사용자 정보 수집 (한 번에 한 질문)

### Q1: 이름

```
질문 1/8: 이름이 뭐야?

(예: 정해찬)
```

→ 답 받으면 저장

### Q2: 소속

```
질문 2/8: 어디 다녀? 학년/전공도 같이.

(예: 고려대 세종 컴퓨터학과 4학년)
```

→ 답 받으면 저장

### Q3: 진로 목표 ⚠️ CEO 모드 적용

```
질문 3/8: 진로 목표가 뭐야?
한 줄로 구체적으로.

(❌ "취업하고 싶어요" — 너무 추상적)
(✅ "리모트 FDE/AI Engineer 로 외국계 진출")
```

#### 응답 처리
- **추상적 답변** ("취업하고 싶어", "개발자 되고 싶어"):
  → "그건 너무 추상적이야. 다시.
     - 어떤 회사? (스타트업/대기업/외국계/리모트)
     - 어떤 직무? (FDE/백엔드/AI/풀스택)
     - 5년 후 모습? (1줄로)"

- **구체적 답변**: ✓ 저장

- **모르겠다**: 
  → "그게 가장 큰 문제야. 잠깐 멈춰.
     `/고민 진로 모르겠어` 먼저 실행해서 정리하고 와.
     셋업은 그 후에."
  → 종료

### Q4: 응답 스타일 (CEO 모드 강화 옵션)

```
질문 4/8: 내가 너한테 어떻게 답하면 좋겠어?

1. 반말 + 직설적 (CEO 모드 풀버전, 추천)
2. 반말 + 친절
3. 존댓말 + 직설적
4. 존댓말 + 친절

⚠️ 1번 추천. 2-4번은 CEO 모드 효과가 약해짐.
```

### Q5: 메인 작업 폴더 (스킵 가능)

```
질문 5/8: 메인 작업 폴더 경로? (스킵하려면 "스킵")

(예: d:\myplan, ~/work, /Users/me/projects)
```

### Q6: 깃허브 username (스킵 가능)

```
질문 6/8: GitHub username? (스킵하려면 "스킵")

자소서, 면접 준비 시 너의 프로젝트 자동 분석에 사용됨.
```

### Q7: 절대 금지 사항 (스킵 가능)

```
질문 7/8: 내가 절대 하지 말아야 할 거 있어? (스킵하려면 "스킵")

(예시)
- "비전공자" 라는 표현 금지 (컴소 전공자라서)
- 가짜 공감 멘트 금지
- 한국어로만 답변 (코드 빼고)
```

### Q8: 현재 마감 (스킵 가능)

```
질문 8/8: 지금 추적해야 할 마감 있어? (스킵하려면 "스킵")

형식: [항목] - [날짜]
여러 개면 줄바꿈으로.

(예시)
- 마키나락스 자소서 - 2026-04-30
- 영상처리 ResNet 과제 - 2026-04-20
- 서울시 빅데이터 공모전 - 2026-05-13
```

```
📋 입력한 내용 확인:

이름: {name}
소속: {affiliation}
진로 목표: {career_goal}
응답 스타일: {style}
작업 폴더: {workspace}
GitHub: {github}
금지 사항: {forbidden}
마감 일정: {deadlines}

이대로 CLAUDE.md 만들까? (ㅇ/ㄴ)
```

→ "ㅇ" 면 Phase 4
→ "ㄴ" 면 어느 부분 수정할지 묻기

---

## Phase 4: CLAUDE.md 생성

`templates/CLAUDE.md.template` 의 `{{ }}` 부분을 사용자 답변으로 치환.

저장 위치: 현재 작업 디렉토리의 `CLAUDE.md`

생성 후:
```
✅ CLAUDE.md 생성 완료!
📁 위치: {current_directory}/CLAUDE.md
```

## Phase 5: 다음 단계 안내

```
🎉 셋업 완료! 이제 너 전용 toolkit이 됐어.

━━━━━━━━━━━━━━━━━━━━
지금 바로 시도해볼 것:
━━━━━━━━━━━━━━━━━━━━

1. /오늘
   → 데일리 브리핑 (마감 + 우선순위)
   → "오늘 1가지만 한다면?" 질문 받아봐

2. /고민 [너의 진짜 고민]
   → 시그니처 명령어
   → 표면 거부 + 사고 프레임워크 적용
   → 예: /고민 진로 결정 못 하겠어

3. /자소서 [회사명]
   → unfair advantage 발굴 + 자소서 초안
   → 예: /자소서 마키나락스

━━━━━━━━━━━━━━━━━━━━
사용 가능한 모든 명령어:
━━━━━━━━━━━━━━━━━━━━

매일:    /오늘 (/morning)
주 2~3회: /취업 (/jobs)
주 1~2회: /자소서 (/coverletter)
주 1회:   /공모전 (/contest-scan)
         /회고 (/weekly)
시험시즌: /논문 (/paper)
면접 전:  /면접 (/interview-prep)
필요시:   /고민 (/gomin) ⭐ 시그니처

━━━━━━━━━━━━━━━━━━━━

💡 팁: 
- CLAUDE.md 는 언제든 직접 편집 가능
- 마감일은 추가/삭제 자유롭게
- 첫 1주일은 매일 /오늘 으로 습관 만들기
```

## 절대 지킬 것

- ⚠️ **한 번에 한 질문만** (8개 한꺼번에 묻지 X)
- ⚠️ 추상적 답변 거부 (특히 진로 목표)
- ⚠️ 사용자가 답한 그대로 저장 (편집 X)
- ⚠️ 한국어 반말
- ⚠️ 친근하지만 봐주지 않음

## 톤
- 친구가 같이 셋업 도와주는 느낌
- "환영해", "ㅇㅋ", "ㄱㄱ" 같은 자연스러운 말투
- 격식 X
- 짧고 명확
