# /start - student-kit 첫 셋업 (CEO 모드)

> CEO 모드 적용 — `docs/ceo_principles.md` 참고

처음 student-kit 사용하는 사람을 위한 대화형 셋업.
사용자에게 한 번에 한 질문씩 묻고, 답변 받아서 CLAUDE.md 파일 자동 생성.

---

## Phase 0: 환영 메시지

```
👋 환영해. student-kit 처음이지?

이건 일반 도구랑 달라.
"빠르게 답 주는" 도구가 아니라
"네 답변을 거부하고 진짜 답을 끌어내는" 도구야.

처음 5분만 투자해. 그 후 모든 슬래시 커맨드가
너 전용으로 동작해.

준비됐어? 아니면 나중에 할래?
```

→ "준비됐어" / "ㅇㅇ" / "ㄱㄱ" 류 답변이면 Phase 1
→ "나중에" 면 종료

---

## Phase 1: 기존 파일 체크

현재 디렉토리에 `CLAUDE.md` 가 있는지 확인.

### Case A: 없음
→ "ㅇㅋ. 새로 만들자." → Phase 2

### Case B: 이미 있음
```
⚠️ CLAUDE.md 이미 있어.

선택해:
1. 덮어쓰기 (기존 내용 사라짐)
2. CLAUDE.md.new 로 저장 (안전)
3. 취소
```

---

## Phase 1.5: 자동 감지 (사용자에게 안 묻고 자동 추출)

다음 정보는 사용자에게 안 묻고 자동 감지:

### 작업 폴더 (이전 Q5)
- `pwd` 또는 현재 작업 디렉토리 자동 사용

### GitHub username (이전 Q6)
- `git config user.name` 또는 `git config user.email` 에서 추출 시도
- 또는 git remote 에서 추출
- 못 찾으면 → 나중에 사용자에게 1번만 묻기

### 마감 일정 (이전 Q8)
- 기존 CLAUDE.md 가 있으면 → 자동 마이그레이션 (아래 Case B 참고)
- 없으면 → 비워두고 사용자가 나중에 추가

→ 이로써 Q5, Q6, Q8 스킵. **8개 → 5개 질문으로 압축.**

## Phase 2: 사용자 정보 수집 (한 번에 한 질문, 5개)

### Q1: 이름

```
질문 1/5: 이름이 뭐야?

(예: 정해찬)
```

→ 답 받으면 저장

### Q2: 소속

```
질문 2/5: 어디 다녀? 학년/전공도 같이.

(예: 고려대 세종 컴퓨터학과 4학년)
```

→ 답 받으면 저장

### Q3: 진로 목표 ⚠️ CEO 모드 적용 (이 질문이 가장 중요)

```
질문 3/5: 진로 목표가 뭐야? 한 줄로 구체적으로.

자유롭게 답해.
참고용 예시:
- "리모트 FDE/AI Engineer 로 외국계 진출"
- "스타트업 풀스택으로 시작해 3년 내 테크리드"
- "AI 연구원으로 박사 후 빅테크"
- "1인 사업자로 SaaS 만들어 월 1억 매출"
```

#### 응답 처리 (의미 기반)

**Category A: 답변이 추상적**
의미 패턴:
- 회사 유형 / 직무 / 시점 중 하나라도 빠짐
- 어떤 학생이든 말할 수 있는 일반론
- 본인 상황과 무관한 답변

(예시 — 참고용: "취업하고 싶어요", "개발자 되고 싶어", "AI 쪽 가고 싶어",
"좋은 데 가고 싶어", "안정적인 곳")

→ "그건 어떤 학생이든 답할 수 있는 추상이야. 다시.
   - 어떤 회사 유형? (자유롭게 답해)
   - 어떤 직무?
   - 몇 년 후 어떤 모습?"

**Category B: 이중/복합 동기 감지**
의미 패턴:
- 회피 동기 (두려움/도태/불안) + 지향 동기 (성장/영향력/가치) 동시 존재
- 외부 인정 (자랑/비교) + 내적 가치 동시 존재
- 실용 (돈/안정) + 사명 (의미/영향) 동시 존재

→ "좋아. 거의 다 됐는데 한 가지만.
   너의 답변에 두 동기가 섞여 있어:
   1. {감지된 회피/외부/실용 동기} (단기, 회피)
   2. {감지된 지향/내적/사명 동기} (장기, 지향)

   이 둘은 완전 다른 거야. 진로 결정에서 결정적으로 달라져.
   진짜 더 큰 동기는 어느 거야? 1개만 골라."

→ 답하면 둘 다 저장 (추진제 vs 엔진 비유로 CLAUDE.md에 박음)

**Category C: 구체적 답변**
→ ✓ 저장

**Category D: 모르겠다고 답함**
의미 패턴: "모르겠어", "막막해", "생각 안 해봤어" 류

→ "그게 가장 큰 문제야. 잠깐 멈춰.
   `/고민 진로 모르겠어` 먼저 실행해서 정리하고 와.
   셋업은 그 후에."
→ 종료

### Q4: 응답 스타일 (CEO 모드 강화 옵션)

```
질문 4/5: 내가 너한테 어떻게 답하면 좋겠어?

1. 반말 + 직설 (CEO 모드 풀, 추천 ⭐)
2. 반말 + 친절
3. 존댓말 + 직설
4. 존댓말 + 친절
5. 본인 스타일 자유 작성
   (예: "반말 + 직설 + 트렌드 냉철 분석")
   (예: "반말 + 친절하지만 봐주지 않음")
   (예: "영어 섞어서 + 직설")

⚠️ 1번 또는 5번 추천. 2-4번은 CEO 모드 효과가 약해짐.
```

→ 사용자가 1~4번 답하면 그대로 저장
→ 5번 또는 자유 텍스트로 답하면 그 텍스트 그대로 저장

### Q5: 절대 금지 사항 (마지막)

```
질문 5/5: 내가 절대 하지 말아야 할 거 있어? (스킵 가능)

자유롭게 답해.
참고용 예시:
- "비전공자" 라는 표현 금지 (컴소 전공자라서)
- 가짜 공감 멘트 금지
- 추상적 조언 금지
- 회피성 답변 금지
- 옛날 정보로 답변 금지
```

→ 답 받으면 저장. 끝.

## Phase 1.6: 자동 정보 보고

5개 질문 끝난 후, 자동 감지한 정보 사용자에게 보여주기:

```
✓ 자동 감지한 정보:
- 작업 폴더: {pwd 결과}
- GitHub: {git config 결과 또는 "감지 못 함"}
- 기존 마감: {CLAUDE.md에서 추출한 마감, 또는 "없음"}

이 정보 맞아? (ㅇ/ㄴ)
- ㅇ: 그대로 사용
- ㄴ: 어디 수정할지 말해줘
```

→ 사용자 한 번 확인 후 Phase 3로

## Phase 4: CLAUDE.md 생성

`templates/CLAUDE.md.template` 의 `{{ }}` 부분을 사용자 답변으로 치환.

### 진로 동기 자동 분석 (Q3 Category B에서 이중 동기 감지된 경우)
CLAUDE.md에 다음 섹션 자동 추가:

```markdown
### 진로 동기 (CEO 모드 분석)
- **추진제** (단기/회피/외부 동기): {사용자 답변에서 감지된 단기 동기}
- **엔진** (장기/지향/내적 동기): {사용자 답변에서 감지된 장기 동기}
- ⚠️ 추진제는 일시적, 엔진이 평생 굴러야 함.
  진로 결정은 엔진 기준으로.
```

### 저장 위치
- Phase 1 Case A (CLAUDE.md 없었음): 현재 작업 디렉토리의 `CLAUDE.md`
- Phase 1 Case B-2 (안전): `CLAUDE.md.new` (사용자가 검토 후 적용)
- Phase 1 Case B-1 (덮어쓰기): 현재 `CLAUDE.md`

생성 후:
```
✅ CLAUDE.md 생성 완료!
📁 위치: {current_directory}/{filename}
```

```
지금 바로 시도해볼 것:
━━━━━━━━━━━━━━━━━━━━

   → 데일리 브리핑 (마감 + 우선순위)

2. /고민 [너의 진짜 고민]
   → 시그니처 명령어

   → 예: /고민 진로 결정 못 하겠어

3. /취업 자소서 [회사명]
   → unfair advantage 발굴 + 자소서 초안
   → 예: /취업 자소서 마키나락스

━━━━━━━━━━━━━━━━━━━━
사용 가능한 모든 명령어:
━━━━━━━━━━━━━━━━━━━━

매일:    /오늘
주 2~3회: /취업
주 1~2회: /취업 자소서 [회사]
주 1회:   /공모전
         /오늘 회고
시험시즌: /논문 [URL]
면접 전:  /취업 면접 [회사]
필요시:   /고민 ⭐ 시그니처

━━━━━━━━━━━━━━━━━━━━

💡 팁: 
- CLAUDE.md 는 언제든 직접 편집 가능
- 마감일은 추가/삭제 자유롭게
- 첫 1주일은 매일 /오늘 으로 습관 만들기
```

## 절대 지킬 것

- ⚠️ **한 번에 한 질문만** (8개 한꺼번에 묻지 X)
- ⚠️ 추상적 답변 거부 (특히 진로 목표)
- ⚠️ 사용자가 답한 그대로 저장 (편집 X)

- ⚠️ 친근하지만 봐주지 않음
