# /today (/오늘) - 시간 관련 모든 작업 통합

> CEO 모드 적용 — `docs/ceo_principles.md` 참고

---

## Sub-Action 파싱

`$ARGUMENTS` 첫 단어로 모드 결정:

| 입력 | 모드 | 설명 |
|------|------|------|
| (없음) | **데일리** | 오늘 작업 브리핑 |
| `회고` | **주간 회고** | 지난 1주일 회고 + 다음 주 계획 |
| `진척` | **월간 진척** | 지난 30일 진척도 추적 |
| `빠르게` / `fast` | **빠른 모드** | CEO 질문 스킵, 결과만 |

> 💡 일요일이면 자동으로 `회고` 모드 추천 ("/오늘 회고 어때?")
> 💡 매월 1일이면 자동으로 `진척` 모드 추천

## 📅 모드 1: 데일리 (default)

### Phase 0: CEO Challenge (질문 1개)

> **"오늘 딱 1가지만 한다면 뭐가 가장 의미 있어?"**
>
> 나머지는 다 부수적이야.
> 이거 못 정하면 오늘도 그냥 흘러갈 거야.

**응답 처리 (의미 기반)**:
- **명확한 1가지** → ✓ Phase 1로
- **여러 개 나열** → "다 중요해 보여? 정확히 하나만 골라."
- **모르겠음** → "그게 진짜 문제야. plan.md 같이 보면서 찾자."
- **빠른 모드 키워드** → CEO 스킵, Phase 1 바로

### Phase 1: 컨텍스트 로드 (캐시 우선)
- `CLAUDE.md` 컨텍스트에 있으면 재읽기 X
- 없으면 Read (사용자 정보 + 마감)
- `plan.md`, `todo.md`, `deadlines.md` 는 Grep으로 마감/날짜 부분만

### Phase 2: 마감 + 어제 작업 (병렬)
동시 실행:
- 마감 정보 추출 (오늘 기준 7일 이내)
- `git log --since="yesterday" --oneline`

### Phase 3: 출력 (압축)
```
☀️ 좋은 아침! YYYY-MM-DD (요일)

🎯 오늘의 1가지
→ {Phase 0 답변}

🚨 임박 마감 (7일 이내)
| 마감일 | D-day | 항목 |
|--------|-------|------|

📝 어제 작업
- {1~3줄}

📋 우선순위 Top 3
1. ⭐ {1가지 답변} - [예상 시간]
2. {2순위}
3. {3순위}

🚀 지금 시작
→ {30분~1시간 작업 1개}
```

## 📊 모드 2: 회고 (`/오늘 회고`)

### Phase 0: CEO Challenge (질문 2개)

> **Q1**: "이번 주 의미 있는 일 1개 했어?"
> 이름 말할 수 있는 거. 못 하면 → 이번 주 망함. 인정하고 다음으로.

> **Q2**: "다음 주에 또 같은 일 할 거야?"
> 변화 없으면 회고 의미 없어. 변화 1개라도 정해.

**응답 처리 (의미 기반)**:
- **명확한 1개 + 변화 1개** → ✓ Phase 1로
- **활동 못 떠올림** → "그것 자체가 인사이트야. 회고에 그대로 적자."
- **변화 없을 듯** → "다음 주 똑같은 회고 쓸 거야. 변화 1개만이라도."

### Phase 1: 활동 수집 (병렬)
- `git log --since="1 week ago" --all --oneline`
- 주요 프로젝트 폴더 git status

### Phase 2: 카테고리 분류
- 🎓 학업 / 💼 취업 / 🏆 대회 / 💻 프로젝트 / 📝 콘텐츠 / 📚 학습

### Phase 3: 회고 작성

파일명: `weekly_YYYY-MM-DD.md`

```markdown
# 주간 회고: YYYY-MM-DD ~ YYYY-MM-DD

## ⭐ 의미 있는 일 (Phase 0)
→ {사용자 답변}

## 🎯 카테고리별 활동
[git log 기반]

## 📊 통계
- 총 commits: N개
- 작업 프로젝트: N개
- 자소서 작성: N개

## 💡 잘한 점
## 😅 아쉬운 점

## 🔄 다음 주 변화 (Phase 0)
→ {사용자 답변}

## 🚀 다음 주 계획
```

## 📈 모드 3: 진척 (`/오늘 진척`)

### Phase 0: CEO Challenge (질문 1개)

> **"지난 한 달, 비전에 한 발짝이라도 가까워졌어?"**
>
> 정량 답변 OK. 정성도 OK.
> 모르겠으면 → 진척 측정 시스템 없는 거. 그게 답이야.

### Phase 1: 데이터 수집 (병렬)
- `git log --since="30 days ago" --all --oneline | wc -l`
- 자소서 파일 카운트: `glob coverletter_*.md`
- 프로젝트 ship 카운트
- /고민 사용 기록 (가능하면)

### Phase 2: 비전 정렬 체크
- `CLAUDE.md` 의 진로 목표 읽기
- 지난 30일 활동이 그 목표에 정렬되는지 평가

### Phase 3: 출력
```
📊 지난 30일 진척도

🎯 너의 비전: {CLAUDE.md 진로 목표}

📈 행동 메트릭
- Commits: N개 (지난 달 대비 ±%)
- 자소서 작성: N개
- 잡 지원: N개 / 통과: N개 (%)
- 프로젝트 ship: N개
- /고민 사용: N회

🧭 비전 정렬도
- ✅ 정렬됨: {활동 목록}
- ⚠️ 무관: {활동 목록}
- ❌ 비전 반대 방향: {있으면}

💡 핵심 통찰
1. {데이터 기반 통찰 1}
2. {데이터 기반 통찰 2}
3. {데이터 기반 통찰 3}

🚨 경고 신호 (있으면)
- 의심 ↑ 행동 ↓
- 비전 무관 활동 비율 50% 초과
- /고민만 많고 ship 없음

🎯 다음 30일 1순위
→ {정렬도 + 메트릭 기반 추천}
```

## ⚠️ 절대 금지

- ❌ 3개 모드를 한 번에 다 보여주기 (인지 과부하)
- ❌ 빠른 모드인데 CEO 질문 추가
- ❌ 이미 읽은 파일 재읽기
- ❌ 일반론 ("열심히 해", "잘하고 있어")
- ❌ 한국어 외 답변
