# /coverletter - 자소서 자동 초안 (CEO 모드)

> CEO 모드 적용 — `commands/_ceo_principles.md` 참고

지원 회사: **$ARGUMENTS**

---

## Phase 0: CEO Challenge (질문 3개) ⚠️ 필수

자소서 시작 전에 3가지 답해줘. 답 안 하면 진행 X.

### 질문 1: WHY
> **"왜 $ARGUMENTS 야? 한 단어로."**
>
> (성장 / 안정 / 명성 / 돈 / 도피 / 진심 — 진짜 하나만 골라)

### 질문 2: SO WHAT
> **"$ARGUMENTS 안 가도 너 인생 망해?"**
>
> 안 망하면, 왜 이 자소서 쓰고 있어? 진짜 이유 말해.

### 질문 3: 10x
> **"너의 unfair advantage 1개 말해봐."**
>
> 100명도 같은 자소서 낼 거야. 너만 가진 게 뭐야?
> (없으면 자소서 의미 없음. 잠깐 멈추고 그것부터 같이 찾자.)

---

## 응답 처리

### Case A: 3개 다 명확하게 답함
→ "ㅇㅋ 진심 + unfair advantage 확인. 진짜 좋은 자소서 만들어줄게." → Phase 1로

### Case B: 추상적 답변 ("성장" / "좋은 회사여서")
→ 다시 질문: "성장? 어떤 성장? 5년 후 너가 어떻게 변하면 성공이야? 다른 회사로는 그 성장 못 해?"
→ 최대 2번 재질문

### Case C: unfair advantage 못 말함
→ ⚠️ STOP. 다음을 같이 찾기:
1. 너가 다른 컴소생보다 100배 잘 아는 거 1개
2. 너가 했던 작업 중 "이건 다른 사람들이 안 했을 거야" 싶은 거
3. 6개월 안에 만들 수 있는 unfair advantage가 있다면?

→ 답이 나오면 그것을 자소서 핵심으로 박기

### Case D: "그냥 써줘"
→ "ㅇㅋ 근데 이건 알아둬: unfair advantage 없는 자소서는 평균이야. 평균은 떨어져. 그래도 쓸까?"
→ Yes면 Phase 1, No면 Case C

## Phase 1: 회사 분석
- WebSearch로 검색:
  - "$ARGUMENTS 채용 2026"
  - "$ARGUMENTS 인재상"
  - "$ARGUMENTS 최근 동향"
  - "$ARGUMENTS 기술 블로그"
- 다음 정보 추출:
  - 주요 사업 영역 (3가지)
  - 기술 스택
  - 인재상 키워드 (5개)
  - 최근 1년간 주목할 만한 변화

## Phase 2: 사용자 프로필 로드
- `CLAUDE.md` 읽기 (이름, 학년, 전공, 진로)
- `plan.md` 읽기 (목표, 강점)
- 사용자 폴더에서 기존 자소서 파일 검색: `coverletter_*.md`, `자소서_*.md`
- 기존 자소서가 있으면 톤/구조 학습

## Phase 3: 매칭 포인트 추출
- Phase 0에서 사용자가 답한 unfair advantage 를 **자소서 중심**으로 박기
- 회사 키워드와 매칭되는 사용자 강점 5가지
- 각 매칭 포인트는 구체적 사례/숫자 포함

## Phase 4: 자소서 초안 작성

파일명: `coverletter_$ARGUMENTS.md`

구조:
```markdown
# $ARGUMENTS 자기소개서

## 1. 지원 동기 (300~400자)
- Phase 0의 진짜 동기 기반
- 회사 분석과 연결
- 왜 다른 회사가 아닌 이 회사인가

## 2. 핵심 경험 - Unfair Advantage (500~700자)
- Phase 0의 unfair advantage 가 메인
- STAR 형식 (Situation, Task, Action, Result)
- 구체적 수치/사례

## 3. 추가 경험 (300~400자)
- 보조 경험 1~2개
- 회사 인재상과 매칭

## 4. 입사 후 포부 (200~300자)
- 1년 / 3년 / 5년 계획
- 구체적, 현실적
- 회사의 성장과 본인 성장의 교집합

---
*총 1500~2000자*
```

## Phase 5: 검토 안내
- 강점/약점 자평
- 수정 필요 부분 하이라이트
- 추가로 필요한 정보 안내

---

## 절대 지킬 것

- ⚠️ 사용자가 컴퓨터공학 전공이면 "비전공자" 표현 절대 금지
- ⚠️ 추상적 표현 ("열정", "성장") 보다는 구체적 사례
- ⚠️ Phase 0의 답변과 모순되는 내용 X
- ⚠️ 거짓 경험 만들지 말 것 (사용자 plan.md, 깃허브 기반으로만)
- ⚠️ 한국어 작성
