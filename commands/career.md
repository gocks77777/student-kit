## 🔍 모드 1: 잡 헌팅 (default)

### Phase 0: CEO Challenge (질문 3개)

> **Q1 (WHY)**: "너 진짜 취업하고 싶어? 부모님/동기 때문 아니야?"
>
> **Q2 (강제 1순위)**: "1순위 회사 유형 1개만 골라."
> 자유롭게 답해 — 카테고리 정해진 거 없음.
> (참고용: 리모트 / 외국계 / 한국 대기업 / 한국 스타트업 / 프리랜서 / 정부기관 / 연구소 / 본인 카테고리)
>
> **Q3 (인버전)**: "합격해서 1년 안에 그만둘 회사야? 그럼 왜 지원해?"

### 응답 처리 (의미 기반)
- **Category A: 3개 다 명확** → ✓ Phase 1로
- **Category B: 우선순위 모호** (예: "다 괜찮아", "딱히 상관없어") → "그게 문제야. 1개만 골라."
- **Category C: 회피성** → "그래도 진행할까? 우선순위 없는 검색은 시간 낭비"

### Phase 1: 사용자 프로필 (캐시 우선)
- `CLAUDE.md` 컨텍스트 있으면 재읽기 X
- 진로 목표 + 기술 스택 추출
- Phase 0 답변을 검색 우선순위로 설정

### Phase 2: Job Finder + 검색 (병렬)
- 사용자 Job Finder 스크립트 정의돼 있으면 실행
- WebSearch 동시 실행:
  - "{Phase 0 카테고리} {직무} 채용 2026"
  - 한국 사이트: 사람인, 잡코리아, 원티드, 점핏
  - 해외 사이트: LinkedIn, We Work Remotely

### Phase 3: 매칭 + Top 5 출력
점수 기준: 기술 매칭 (4) + 경력 (3) + 지역 (2) + 인지도 (1) + Phase 0 매칭 가중치 (+2)

```
| 순위 | 회사 | 직무 | 점수 | 마감 | 추천 |
```

### Phase 4: 액션 추천
Top 1~2 회사에 대해:
- "지원할만 함" / "검토 필요" / "패스 추천"
- 지원 결정하면 → "/취업 자소서 [회사명]" 추천

## ✍️ 모드 2: 자소서 작성 (`/취업 자소서 [회사]`)

### Phase 0: CEO Challenge (질문 3개)

> **Q1 (WHY)**: "왜 [회사]야? 너의 진짜 동기를 한 줄로."
> 자유롭게 답해. 카테고리 정해진 거 없음.
>
> **Q2 (SO WHAT)**: "[회사] 못 가면 인생 망해? 망하든 안 망하든, 왜 이 회사야?"
>
> **Q3 (10x)**: "너의 unfair advantage 1개. 100명도 같은 자소서 낼 거야. 너만 가진 게 뭐야?"

### 응답 처리 (의미 기반)
- **Category A**: 3개 다 진심 + 구체적 → ✓ Phase 1로
- **Category B**: 추상적 → 다시 질문 (1회만)
- **Category C**: unfair advantage 못 말함 → STOP. 같이 찾기:
  1. 다른 컴소생보다 100배 잘 아는 거 1개
  2. "이건 다른 사람들이 안 했을 거야" 싶은 거
  3. 6개월 안에 만들 수 있는 unfair advantage
- **Category D**: "그냥 써줘" → 경고 후 진행

### Phase 1: 회사 분석 (병렬 검색)
WebSearch 동시 실행:
- "[회사] 채용 2026"
- "[회사] 인재상"
- "[회사] 최근 동향"
- "[회사] 기술 블로그"

> 💰 토큰 절약: 1세션에 같은 회사 검색했으면 캐시 사용

### Phase 2: 사용자 프로필 (캐시 우선)
- `CLAUDE.md` + Glob `coverletter_*.md` (가장 최근 1개)

### Phase 3: 자소서 작성 (1차)
파일명: `coverletter_[회사].md`

구조:
- 1. 지원 동기 (Phase 0 동기 기반, 300~400자)
- 2. 핵심 경험 - Unfair Advantage (Phase 0의 advantage 메인, 500~700자)
- 3. 추가 경험 (300~400자)
- 4. 입사 후 포부 (200~300자)

### Phase 4: Bar-Raiser (자동 검증) ⭐
1차 초안 후 자동 자체 검증:

1. **Future Self**: 1년 후 너가 이거 보면 자랑스러울까?
2. **Top 1% 비교**: 100명 중 1등 자소서랑 비교하면?
3. **Phase 0 일관성**: Phase 0 답변과 일치하는가?
4. **숨은 약점**: 어디가 약해?

→ 4개 다 통과 → 출력
→ 1개라도 미통과 → 자동 재작업 (최대 2회)
→ 3번째도 미통과 → 사용자에게 "왜 안 되는지" 진단

## 🎤 모드 3: 면접 대비 (`/취업 면접 [회사]`)

### Phase 0: CEO Challenge (질문 3개)

> **Q1 (WHY)**: "왜 이 회사 합격해야 해? 합격해도 행복할 거 같아?"
>
> **Q2 (약점 진단)**: "면접에서 제일 두려운 질문 1개는?"
> "왜 두려워?" 가 너의 진짜 약점.
>
> **Q3 (인버전)**: "떨어지면 어떻게 돼? 객관적으로 말해. 인생 안 망해."

### 응답 처리
- **3개 다 명확** → Phase 1로
- **두려운 질문 회피** → "그게 답이야. 그 영역부터 준비하자."
- **합격에 절박** (예: "무조건 합격") → "절박할수록 떨어져. 워라밸 마인드로."

### Phase 1: 회사 + 직무 조사 (병렬)
WebSearch 동시:
- "[회사] 면접 후기" (잡플래닛, 블라인드)
- "[회사] 기술 블로그"
- "[회사] 채용 페이지 우대사항"
- "[회사] 최근 1년 동향"

### Phase 2: 사용자 프로필 + Phase 0 약점 영역 표시

### Phase 3: 예상 질문 카테고리별 생성
- A. 인성/동기 (3개)
- B. 기술 (5개)
- C. 프로젝트 (5개)
- **D. ⚠️ 약점 질문** (Phase 0 두려운 질문 + 유사 3~5개)
- E. 시나리오 (2개)

### Phase 4: 답변 가이드 + 출력
파일명: `interview_prep_[회사].md`

각 질문:
- STAR 형식 템플릿
- 사용자 프로젝트 기반 예시 답변
- 주의 포인트
- 꼬리 질문 예상

### Phase 5: Bar-Raiser
- 약점 질문 답변이 진짜 약점을 커버하는가?
- 거짓말 있는가?
- "떨어져도 OK" 마인드 명시했는가?

## ⭐ 모드 4: 풀준비 (`/취업 풀준비 [회사]`)

> 🔥 student-kit의 시그니처 Pipeline 명령어
> 1번 명령어로 회사 분석 → 자소서 → 면접 → 검증 → 패키지

### Phase 0: 메가 CEO Challenge (질문 5개)

> 잠깐. 풀준비는 진짜 비싼 작업이야. 5가지 답해.
>
> 1. 왜 [회사]야? 한 줄.
> 2. 못 가면 인생 망해? 망하든 안 망하든, 왜 이 회사야?
> 3. 너의 unfair advantage 1개.
> 4. 면접 떨어지면 어떻게 돼?
> 5. 1년 후 너가 이 회사 다니고 있다면, 어떻게 살고 있어?

→ 5개 다 답해야 진행. 1개라도 모호하면 거부.

### Phase 1: 회사 풀 분석 (병렬 검색 - 가장 비싼 단계)
WebSearch 동시:
- "[회사] 채용 2026"
- "[회사] 인재상"
- "[회사] 최근 동향"
- "[회사] 기술 블로그"
- "[회사] 면접 후기"
- "[회사] 연봉 / 복지"

→ 결과를 메모리에 캐시 (다음 Phase에서 재사용)

### Phase 2: 자소서 작성 (Bar-Raiser 포함)
- 모드 2 자소서 워크플로우 실행
- Phase 0 답변 5개를 모두 활용
- Bar-Raiser 자동 검증

### Phase 3: 면접 대비 (Bar-Raiser 포함)
- 모드 3 면접 워크플로우 실행
- Phase 0의 약점 질문 활용

### Phase 4: 통합 패키지
파일명: `career_package_[회사].md`

```markdown
# [회사] 풀 준비 패키지

## 회사 요약
[Phase 1 결과]

## 자소서 (coverletter_[회사].md 참조)
[하이라이트만]

## 면접 (interview_prep_[회사].md 참조)
[하이라이트만]

## 체크리스트
- [ ] 자소서 1주일 후 재검토
- [ ] 면접 답변 3번 소리내어 연습
- [ ] 회사 최근 뉴스 면접 전날 확인
- [ ] 떨어져도 OK 마인드 준비
- [ ] 입사 후 1년 계획 (Phase 0 Q5)

## 다음 액션 (24시간)
→ {가장 작은 실행 1개}
```

### Phase 5: 메타 Bar-Raiser
- 자소서 + 면접이 서로 일관되는가?
- Phase 0 5개 답변이 모든 출력에 반영되는가?
- 풀 패키지가 "이 회사 가야 한다" 를 진심으로 설득하는가?

## 📊 모드 5: 진척 (`/취업 진척`)

### Phase 0: CEO Challenge (질문 1개)

> **"지금 취업 잘 돼가? 데이터로 말해봐."**
>
> 데이터 없으면 → "그게 진짜 문제야. 측정 시작하자."

### Phase 1: 데이터 수집
- Glob `coverletter_*.md` → 자소서 카운트
- Glob `interview_prep_*.md` → 면접 준비 카운트
- `git log` 에서 취업 관련 commit 추출
- CLAUDE.md 에서 마감 일정 추출

### Phase 2: 통계 출력
```
📊 취업 활동 (지난 30일)

✍️ 자소서: N개 작성
   - 통과: ?개 / 진행 중: ?개 / 떨어짐: ?개

🎤 면접 준비: N개

📤 잡 지원: N개
   - 답변 받음: ?개
   - 면접 본 거: ?개
   - 합격: ?개

💯 통과율: ?%
   ⚠️ 통과율 < 5% → 자소서 품질 의심
   ⚠️ 답변률 < 10% → 매칭도 의심

🎯 다음 30일 추천
- {데이터 기반}
```

### Phase 3: 인사이트
- 패턴 분석
- 약점 영역 진단
- 다음 액션 추천
