> CEO 모드 적용 — `docs/ceo_principles.md` 참고

---

## Sub-Action

| 입력 | 모드 |
|------|------|
| (없음) | **탐색** (default) |

## 🔍 모드 1: 탐색 (default)

### Phase 0: CEO Challenge (질문 3개)

> **Q1 (인버전)**: "이미 진행 중인 대회 끝냈어?"
> 안 끝났으면 새 거 추가 = 둘 다 망함. 진짜로.

> **Q2 (전제 도전)**: "수상이 목표야 아니면 경험이야?"
> 자유롭게 답해 — 둘 다 원하면 둘 다 못 얻음.
> 참고용:
> - **수상** → 경쟁 적은 대회, 본인 강점 100% 활용
> - **경험** → 새로운 분야 도전, 결과 신경 X
> - **본인 정의** → 예: "포트폴리오용 수상"

> **Q3 (SO WHAT)**: "스펙 1줄 쌓으려고 하는 거야?"
> 그럼 더 ROI 높은 게 있을 수도 있어:
> - 오픈소스 contribution
> - 자기 프로젝트 (깃허브 스타)
> - 블로그 글 (조회수)
> - 인턴 경험

### 응답 처리 (의미 기반)
- **3개 다 명확** → ✓ Phase 1로
- **진행 중인 대회 있음** → "그거 끝내고 와. 새 거 검색은 그 후."
- **목표 모호 / 둘 다 원함** → "1개만 골라. 못 고르면 잠깐 멈춰."

### Phase 1: 사용자 프로필 (캐시 우선)
- `CLAUDE.md` 컨텍스트에 있으면 재읽기 X
- `competition_plan.md` 는 Grep으로 진행 중인 대회만 추출

### Phase 2: 공모전 탐색 (병렬)
WebSearch 동시 실행:
- contestkorea.com (대학생 공모전 1위)
- linkareer.com (링커리어)
- wevity.com (위비티)
- thinkforbl.com
- "AI 공모전 2026", "데이터 분석 공모전 2026", "SW 개발 공모전 대학생"

### Phase 3: 필터링
- ✅ 마감 14일 이상 남음
- ✅ 대학생 참가 가능
- ✅ Phase 0 목표 (수상 vs 경험) 일치
- ❌ 이미 competition_plan.md 에 있는 것

### Phase 4: 평가 점수
적합도(5) + 임팩트(5) + 시간 비용(5) + Phase 0 매칭 가중치(+2~3) = /15

### Phase 5: 출력
```
🏆 새 공모전 발견 (N개)

| 순위 | 대회명 | 마감 | D-day | 적합도 | 임팩트 | 추천 |

📝 추천 1순위: [대회명]
- 왜 추천: ...
- 준비 기간: ...
- 필요 스킬: ...
- 상금/혜택: ...
- 신청 URL: ...

⚠️ 참가확인증 발급이 학교 인정 기준에 맞는지 확인 필요
```

### Phase 6: Bar-Raiser
- 추천 1순위가 Phase 0 답변과 진짜 일치하는가?
- "수상" 목표였는데 경쟁률 너무 높은 거 추천했나?
- "경험" 목표였는데 너무 익숙한 분야 추천했나?

## 📋 모드 2: 신청 (`/공모전 신청 [대회명]`)

### Phase 0: CEO Challenge (질문 1개)

> **"이 대회 진짜 끝까지 할 수 있어? 1주일 안에 포기 안 할 자신?"**
>
> 학생의 30%가 신청만 하고 끝까지 X.
> 진심이면 → 진행. 자신 없으면 → 잠깐 멈춰.

### Phase 1: 대회 정보 수집
- WebSearch "[대회명] 신청 방법"
- WebSearch "[대회명] 제출 서류"
- 마감일, 자격 요건 추출

### Phase 2: 체크리스트 생성
파일명: `contest_apply_[대회명].md`

```markdown
# [대회명] 신청 체크리스트

## 신청 정보
- 신청 마감: YYYY-MM-DD (D-N)
- 작품 마감: YYYY-MM-DD (D-N)
- 신청 URL: ...

## 필수 서류
- [ ] 참가 신청서
- [ ] 재학증명서 (학교 발급)
- [ ] 팀 구성표 (있는 경우)
- [ ] 작품 기획서

## 학교 인정 (참가확인증)
- [ ] 발급 시점 확인 (접수일 vs 발급일)
- [ ] 학교 학점 인정 가능 여부 확인

## 실행 계획
- 1주차: 기획 + 자료 수집
- 2주차: 개발/제작
- 3주차: 검토 + 마무리

## 다음 24시간 액션
→ {가장 작은 1개}
```

## 📊 모드 3: 추적 (`/공모전 추적`)

### Phase 0: CEO Challenge (질문 1개)

> **"진행 중인 대회 몇 개야? 솔직히."**
>
> 3개 이상이면 → ⚠️ 다 망할 가능성. 잠깐 멈춰.

### Phase 1: 데이터 수집
- `competition_plan.md` 읽기 (Grep)
- `contest_apply_*.md` 글로브
- git log 에서 대회 관련 commit

### Phase 2: 출력
```
🏆 진행 중인 공모전

| 대회명 | 마감 | D-day | 진행도 | 상태 |
|--------|------|-------|--------|------|

⚠️ 경고
- 마감 임박 (3일 이내): N개
- 시작 안 한 거: N개
- 진행 중단 의심: N개

🎯 우선순위
1. {D-day 빠르고 진행도 낮은 거}
2. {...}

💡 추천: 동시 진행 3개 초과 → 1개 포기 권장
```

- ⚠️ 한국어 답변
- ⚠️ 캐시 우선
